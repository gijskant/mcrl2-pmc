A number of case studies also made use of the CADP toolset, mainly for
model-checking μ-calculus formulae and visualization purposes. On the list of
case studies done with CADP toolset one finds the following descriptions of such
cases:

* 41: `TCAP Protocol (Transaction Capabilities Procedures) <http://www.inrialpes.fr/vasy/cadp/case-studies/99-c-tcap.html>`_
* 47: `SPLICE Coordination Architecture <http://www.inrialpes.fr/vasy/cadp/case-studies/00-a-splice.html>`_
* 54: `Development of a Verified Erlang Program for Resource Locking <http://www.inrialpes.fr/vasy/cadp/case-studies/01-a-locker.html>`_
* 60: `AIDA (Automatic In-Flight Data Acquisition) System <http://www.inrialpes.fr/vasy/cadp/case-studies/02-a-lynx.html>`_
* 61: `JavaSpaces (TM) Shared Data Space Architecture <http://www.inrialpes.fr/vasy/cadp/case-studies/02-b-javaspaces.html>`_
* 62: `Needham-Schroeder Public Key Authentication Protocol <http://www.inrialpes.fr/vasy/cadp/case-studies/02-c-needham-schroeder.html>`_
* 63: `Replication Features of the Splice Architecture <http://www.inrialpes.fr/vasy/cadp/case-studies/02-d-replication-splice.html>`_
* 68: `Positive Acknowledgement Retransmission (PAR) Protocol <http://www.inrialpes.fr/vasy/cadp/case-studies/03-a-par.html>`_
* 69: `Jackal Distributed Shared Memory Implementation of Java <http://www.inrialpes.fr/vasy/cadp/case-studies/03-b-jackal.html>`_
* 70: `Distributed Data Space Architectures Described in Space Calculus <http://www.inrialpes.fr/vasy/cadp/case-studies/03-c-space-calculus.html>`_
* 71: `Parallel Programs Developed using JavaSpaces (TM) <http://www.inrialpes.fr/vasy/cadp/case-studies/03-d-javaspaces.html>`_
* 81: `Formal Verification of a Fair Payment Protocol <http://www.inrialpes.fr/vasy/cadp/case-studies/04-c-fair-payment.html>`_
* 87: `Verification of a Fair Non-Repudiation Protocol <http://www.inrialpes.fr/vasy/cadp/case-studies/05-b-non-repudation.html>`_
* 89: `Verifying Fault-Tolerant Erlang Programs  <http://www.inrialpes.fr/vasy/cadp/case-studies/05-d-fault-tolerance.html>`_
* 92: `Formal Specification and Verification of a DRM Protocol  <http://www.inrialpes.fr/vasy/cadp/case-studies/06-c-drm.html>`_

A set of benchmark models that includes mCRL2 specifications is the
`BEEM collection <http://anna.fi.muni.cz/models/?page=models-list>`_.
